1. 了解 C/S 结构与 B/S 结构:
    1.1 结构体系
        Client/Server,即客户端/服务器结构;
        Browser/Server,即浏览器/服务器结构;
    1.2 比较
        1.开发和维护成本方面: C/S 结构的开发和维护成本都比 B/S 结构高;
        2.客户端负载: C/S 结构的客户端负载较高;
        3.安全性: C/S 结构的安全性较低;
2. 理解 Web 应用程序的工作原理:
    Web 应用程序大体上分为两种: 静态网站和动态网站;
3. 了解 Web 应用的客户端应用技术:
    3.1 HTML 语言
        客户端技术的基础,主要用于显示网页信息,由浏览器解释执行,不需要编译;
    3.2 CSS
        样式表(Style Sheet)技术,也称之为层叠样式表(Cascading Style Sheet);
        在制作网页时采用 CSS 样式,可以有效地对页面的布局,字体,颜色,背景等效果实现更加精确的控制;
    3.3 Flash
        交互式矢量动画制作技术,可包含动画,音频,视频以及应用程序;
    3.4 客户端脚本技术
        嵌入到 Web 页面中的程序代码,是一种解释性语言,例如: JavaScript 和 VBScript;
4. 了解 Web 应用的服务器端应用技术:
    4.1 CGI
        Common Gateway Interface,即通用网关接口;
        创建动态网页的一种技术,可以使浏览器与服务器之间产生互动关系;
        允许使用不同的语言来编写适合的 CGI 程序,该程序被放在 Web 服务器上运行;
    4.2 ASP
        Active Server Page 是一种使用很广泛的开发动态网站的技术;
        通过在页面代码中嵌入 VBScript 或 JavaScript 脚本语言,来生成动态的内容;
    4.3 PHP
        Personal Home Page,开发动态网页的技术;
        开源的 Web 服务器脚本语言,公认的安全性能;
    4.4 ASP.NET
        ASP.NET 是一种建立动态 Web 应用程序的技术;
        .NET框架的一部分,可以使用任何.NET 兼容的语言来编写 ASP.NET 应用程序;
        使用 VB .NET,C#,J#,ASP.NET 页面(Web Forms)进行编译,可以提供比脚本语言更出色的性能表现;
    4.5 JSP
        Java Server Page,是以 Java 为基础开发的,沿用 Java 强大的 API 功能;
